Project Overview

The goal is to build a multi-role food delivery ecosystem comprising:

Customer App

iOS & Android

Browse restaurants, place orders, make payments, and track deliveries live.

Rider App

iOS & Android

Accept deliveries, navigate via maps, and update delivery status in real time.

Restaurant App

iOS & Android

Manage menu, prices, orders, and availability.

Restaurant Web App

Web

Manage menu, orders, and restaurant operations through web interface.

Admin Dashboard

Web

Manage restaurants, customers, riders, orders, commissions, and analytics.

Core Features

Customer App

  • User registration and OTP login
  • Search, browse, and filter restaurants
  • Add to cart, apply coupons, and make secure payments
  • Live order tracking on map
  • Push notifications and delivery updates
  • Ratings, reviews, and order history

Rider App

  • Accept / reject deliveries
  • Navigate via Google Maps
  • Update status: Accepted ‚Üí Picked ‚Üí Delivered
  • Daily earnings and delivery history
  • OTP verification at delivery

Restaurant App

  • Receive and manage orders
  • Menu and pricing control
  • Restaurant open/close toggle
  • Daily order reports
  • Order history and insights

Restaurant Web App

  • Menu and pricing management
  • Order management and tracking
  • Restaurant open/close toggle
  • Daily and monthly reports
  • Performance analytics and insights

Admin Dashboard

  • Role-based user management (admins, restaurants, riders)
  • Restaurant approvals and commission management
  • Order tracking and escalation handling
  • Promo code and banner management
  • Advanced analytics dashboard (orders, earnings, user activity)
