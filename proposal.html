Project Milestones

1

Core Monitoring Infrastructure

Timeline: Weeks 1-5

Deliverables:

Goal: Get live network visibility with working backend and basic frontend.

  • Setup of Zabbix and GenieACS (with sample templates for OLTs and CPEs)
  • SNMP and TR-069 data collection
  • Zabbix proxies configured for multi-region polling
  • Device onboarding API (register devices + basic metadata)
  • Basic web interface to view devices and status
2

Alerting & Device Grouping

Timeline: Weeks 6-10

Deliverables:

Goal: Start receiving automated alerts and segment network view by logical units.

  • Alert configuration and management module (via Zabbix API)
  • Device grouping by region/type/template
  • Threshold-based rule engine (warning/critical states)
  • First version of email/webhook alert dispatch
  • Zabbix template auto-mapping based on device type
3

Multi-Tenant SaaS Layer + Admin Portal

Timeline: Weeks 11-15

Deliverables:

Goal: Enable the platform to host multiple ISP clients under isolated environments.

  • Multi-tenant user authentication & isolation
  • ISP client onboarding (tenant creation, role assignment)
  • Admin dashboard with ISP management, activity logs
  • TR-069 parameter UI (basic GenieACS data viewer per CPE)
  • Secure API tokens for customer integrations
4

Polish, QA, Go-Live & Handover

Timeline: Weeks 16-20

Deliverables:

Goal: Ready-to-sell SaaS product with stable backend, UI, and support docs

  • Production-ready deployment
  • QA, UAT, performance tuning
  • Custom client branding (if required)
  • Documentation: API usage, admin training, operational SOPs
  • Optional: onboarding 1‚Äì2 real clients with your support
